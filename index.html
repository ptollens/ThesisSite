Abstract

This project developed and researched the effectiveness of a 3D digital human chat-bot, with the goal of helping students apply to the GMT Masters course. The development goal of the chat-bot is to have an adaptable digital human, which may use several bot frameworks depending on different needs in the future. UnrealEngine's MetaHuman was used to create the digital human, while LangChain with ChatGPT 4o was used to build the dialogue system. Using NVIDIA's Audio2Face, facial animations are able to be generated live using only audio. While the sample size is to small too draw definitive conclusions, participants were more satisfied with their experience with the digital human than using the website, even though the websites were able to answer their questions faster.

Generative A.I. and Retrieval-Augmented Generation (RAG)

To enhance the chat-bot's knowledge and to keep it on topic, Retrieval-Augmented Generation has been used. This has been implemented with the help of LangChain in python. The user's questions gets queried to the local knowledge base, which consists of embedded vector stores. The relevant information, in addition to the user question, is then queried to the LLM, which generates an appropriate response.

Proposed Architecture

The figure below is the chat-bot structure that is implemented for this thesis project. This project had a few key goals, namely: To have a realistic high fidelity digital human, an interactive chat-bot and it should be viable to continue the project in the long-term. To achieve this, the chat-bot consists of a few modular components. For the realistic digital human, UnrealEngine's MetaHuman were used. The facial animations were driven by NVIDIA's Audio2Face, a generative AI model that creates animation from audio only. ChatGPT 4o was used as the LLM provider. Since the structure is modular, it is relativly easy to switch out different components of the chat-bot. Since LangChain works with a variety of LLMs, it is easy to switch the LLM. Audio2Face is responsible for streaming the facial animations, which is not limited to UnrealEngine. One thing to keep in mind, however, is to keep track of which component could be responsible of bottlenecking the system.
